Aegis Documentation

Built-in guide for configuring and operating the Aegis Modbus replication system.

Overview

Aegis is a Modbus data replication and fan-out system designed for industrial telemetry, SCADA integration, and edge data distribution.

Aegis polls Modbus devices and republishes the data through configurable Modbus servers.

Device Setup

Each device in Aegis is made up of three parts:

  • Source — the Modbus device being polled.
  • Reads — the registers or coils to read from the source.
  • Target — the Modbus server that republishes the data.

Source Configuration

Configure the upstream Modbus device to poll.

Field Description
Device Name User-friendly name used to identify the device.
Endpoint TCP address of the Modbus device.
Format: ipaddress:port
Example: 192.168.1.100:502
Unit ID Modbus slave ID of the source device.
Timeout Maximum time to wait for a device response.

Reads Configuration

Configure the polling schedule and register ranges.

Field Description
Function Code Supported function codes:
  • FC1 – Read Coils
  • FC2 – Read Discrete Inputs
  • FC3 – Read Holding Registers
  • FC4 – Read Input Registers
Address Starting Modbus address.
Quantity Number of registers or coils to read.
Interval Polling interval in milliseconds.

Target Configuration

Configure the Modbus replication server that exposes polled data.

Field Description
Port TCP port where the replicated Modbus server will run. Default: 502
Unit ID Modbus unit ID exposed by Aegis.
Status Unit ID Unit ID used for the status memory block.
Status Slot Slot index representing the device inside the status block.
Each device must have a unique Status Slot within the same Status Unit ID.

Device Status

The status panel shows real-time health metrics for each device.

Field Description
Health Indicates current device health state.
Online Shows if the device is responding.
Requests Total Total Modbus requests sent to the device.
Responses Valid Successful responses received.
Timeouts Number of request timeouts.
Seconds in Error How long the device has been in an error state.

Troubleshooting

Device not responding
Check endpoint IP and port.
Invalid responses
Verify correct unit ID and register addresses.
Timeout errors
Increase timeout or verify network connectivity.
